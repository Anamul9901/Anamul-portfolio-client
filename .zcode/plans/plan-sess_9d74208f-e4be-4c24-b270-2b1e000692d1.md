## Hero Section Redesign

### Problem
The current hero has a **dominant 420×525px portrait** (col-span-5 of 12) that overshadows the identity messaging. It says "here's what I look like" first, "here's what I do" second. For a backend-focused engineer, the hierarchy should be flipped.

### New design: Text-forward hero with medium portrait accent

**Layout shift:** 7-column text left → **9-column text left, 3-column portrait right** (portrait shrinks from 420px to ~200px, text area grows from 58% to 75%).

**Image:** ~200px `aspect-square` rounded-xl card with the same tilt + corner accents + floating tag, but positioned as a **right-side accent** — secondary to the text, not the visual anchor.

### New content hierarchy (top → bottom, left column):

1. **Status pill** (keep, already good): "Available for select work — 2026" with ping dot.

2. **Display name** (keep letter stagger): "Anamul Haque" — same animation, same sizing.

3. **Three role badges** (NEW — the key identity addition):
   Styled as mono-label pill chips in a flex row, each with a small icon:
   - `Backend Systems` — FaServer icon, ember tinted
   - `Problem Solver` — FaBrain icon (or FaLightbulb)
   - `Full-Stack` — FaLayerGroup icon
   
   These immediately tell visitors who you are before they read the bio. Compact, scannable, visually distinct.

4. **Role line** (rewritten): 
   "Backend systems architect, competitive programmer & full-stack engineer at **Strvia**, Dhaka."
   Shorter, covers all three selected identities, company name in `--text-0`.

5. **Bio paragraph** (rewritten):
   "Two years shipping high-performance backends and scalable web apps — microservices, multi-vendor platforms, real-time systems. ICPC regionalist, hackathon winner. I care about performance budgets, clean architecture, and the small details that make software feel quiet."

6. **Magnetic CTAs** (keep all 4: Resume, Email, GitHub, LinkedIn) — same styling.

7. **Scroll affordance** (keep): "Scroll ↓"

**Right column (portrait):**
- 200px `aspect-square` `rounded-xl` with TiltCard, corner accents, floating "Backend · Dhaka" tag
- Same scrollY parallax drift (subtle, 20px range)
- Same ember glow behind it
- Shifted slightly lower (align with middle of text block, not top)

### Animation changes
- Status pill: fade-up (keep, delay 0)
- Name: letter stagger (keep, same timing)
- Three role badges: stagger in from left with fade (new, delay ~0.8s, 0.05s apart)
- Role line + bio: fade-up (keep, slightly delayed for badge gap)
- CTAs: fade-up (keep)
- Portrait: same entrance (scale + fade), but smaller so it feels lighter

### File changed
- `src/components/HomeSection/HeroSection.tsx` — single file rewrite, no new dependencies needed (FaServer, FaBrain, FaLayerGroup all already imported elsewhere in the project).
